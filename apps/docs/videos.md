# Docs example videos (`videos/`)

The clips in [`videos/`](videos) are the **output** side of the `<Example>` blocks in the
generator docs — the `pro-visu` config on the left, the video it produces on the right. They're
generated by pro-visu itself from the **FASHION storefront** in [`apps/testing`](../testing), so
the docs double as a live demo of the tool.

They're served through [`next-video`](https://next-video.dev) → **Mux** (adaptive HLS + CDN +
posters). `<Example clip="…" />` looks clips up by slug in
[`components/clips.ts`](components/clips.ts).

> ⚠️ **Only media files belong in `videos/`.** `next-video sync` treats every file there that
> isn't a dotfile / `.json` / `.ts` as a video and uploads it to Mux — that's exactly why this
> doc lives **outside** `videos/` (an earlier `videos/README.md` got uploaded and errored).

## Regenerate the clips

From the repo root, render the curated subset and copy it in:

```bash
# 1. generate (builds + serves apps/testing, captures with pro-visu)
pnpm --filter testing generate --asset type-sans --asset colors-reel --asset lookbook-wall --asset home-vertical

# 2. copy the curated outputs into videos/ (flat names)
cp apps/testing/public/pro-visu/showcase/specimen/type-sans.mp4         apps/docs/videos/type-sans.mp4
cp apps/testing/public/pro-visu/showcase/palette-reel/colors-reel.mp4   apps/docs/videos/colors-reel.mp4
cp apps/testing/public/pro-visu/showcase/wall/lookbook-wall.mp4         apps/docs/videos/lookbook-wall.mp4
cp apps/testing/public/pro-visu/showcase/scroll-reel/home-vertical.mp4  apps/docs/videos/docs-vertical.mp4
```

The fuller set of examples — the `docs-*` clips and the `public/examples/` stills — is generated
from a dedicated config, [`apps/testing/pro-visu/pro-visu.docs.config.ts`](../testing/pro-visu/pro-visu.docs.config.ts):

```bash
pnpm --filter testing exec pro-visu generate --config pro-visu/pro-visu.docs.config.ts
```

Those land in `apps/testing/public/pro-visu/docs/` (its own subfolder + manifest, separate from the
main showcase). Then copy the `docs-*.mp4` into `videos/`, the palette/screenshot PNGs into
`apps/docs/public/examples/`, and convert the (very tall, ~12 MB) full-page desktop screenshot to
JPEG, e.g. `ffmpeg -i docs-shot-desktop-desktop.png -q:v 4 public/examples/docs-shot-desktop.jpg`.

Stills are shown with `<ImageExample src="/examples/…" width={…} height={…} … />` (use the real
pixel dimensions; add `tall` for a scrollable full-page shot). Clips use `<Example clip="…" />`.

To add a new example: render the asset, copy the output in, register a clip in
`components/clips.ts` (or drop a still in `public/examples/`), and reference it with
`<Example clip="…" />` / `<ImageExample … />` in the relevant `.mdx`.

### Replacing a clip that's already on Mux ⚠️

`next-video sync` keys off the `<name>.mp4.json` sibling's `status`, not the file's content —
overwriting an mp4 whose json says `"status":"ready"` is silently ignored, and sync never
deletes Mux assets. To replace one:

```bash
# 1. drop in the new mp4, then forget the old Mux mapping:
rm apps/docs/videos/hero-loop.mp4.json
# 2. re-sync → uploads the new content, fresh .json with a NEW playback id:
pnpm --filter docs sync-videos
# 3. delete the orphaned old asset at https://dashboard.mux.com → Assets
```

`<Example>` picks up the new playback id automatically (it reads the regenerated json).

## Process to Mux (`next-video sync`)

next-video uploads each raw `*.mp4` here to Mux and writes the playback id + poster + blur
placeholder into the adjacent `*.mp4.json` — **that json is what's committed and deployed**;
Vercel never needs the binaries.

```bash
# set MUX_TOKEN_ID / MUX_TOKEN_SECRET (see .env.local.example), then:
pnpm --filter docs sync-videos     # → next-video sync
```

Set the same `MUX_TOKEN_*` vars in the Vercel project env so deploys can process on demand.

> **Source files:** the raw `*.mp4` files are committed so the docs render in `pnpm --filter docs
> dev` without Mux. Once the `*.mp4.json` files carry Mux playback ids, you can stop tracking the
> raw sources (gitignore `videos/*.mp4`) and let Mux serve them — the config that regenerates
> them lives in [`apps/testing/pro-visu/pro-visu.config.ts`](../testing/pro-visu/pro-visu.config.ts).
