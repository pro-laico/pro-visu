title: interaction
description: Realtime recordings of scripted interactions — a synthetic cursor driving the page, or a clip cropped to one component.
---

`interaction` records the page **live** while a scripted sequence plays out — a visible
synthetic cursor moves, clicks, hovers, and types, exactly like a hand-recorded product demo,
but reproducible from config. It always records realtime (interactions and their animations are
inherently time-based) and always emits a single asset.

Give it `actions` (a scripted tour), `focus` (a clip cropped to one component), or both.

## Scripted actions

```ts
options: {
  cursor: { color: "#e91e63" },
  actions: [
    { do: "click", selector: "#menu-button" },
    { do: "hover", selector: ".dropdown a:first-child" },
    { do: "type", selector: "input[type=search]", text: "shoes" },
  ],
}
```

- **`actions: [{ do, selector?, x?, y?, text?, to?, durationMs?, holdMs? }]`** where `do` is
  `"move" | "click" | "hover" | "type" | "scrollTo" | "wait"`. Per action: `durationMs`
  (cursor travel / scroll time) default `700`, `holdMs` (pause after) default `600`.
- **`cursor: { show?, size?, color? }`** — the synthetic cursor. Defaults: `show` `true`,
  `size` `22`, `color` `#0b0b0f`. Set `show: false` for hands-off UI-state clips (e.g. media
  wall tiles).

## Element focus

Capture a single component — scroll it into view, optionally trigger it, hold, and crop the
output to its box:

```ts
options: {
  focus: {
    selector: "#pricing-card",
    padding: 32,
    actions: [{ do: "click", selector: "#billing-annual" }],
    holdMs: 2500,
  },
}
```

- **`focus: { selector, padding?, actions?, holdMs? }`** — defaults: `padding` `24` px,
  `holdMs` `2000` ms. The crop box is measured **after** any trigger actions run, so an
  expanded dropdown/drawer stays in frame.
