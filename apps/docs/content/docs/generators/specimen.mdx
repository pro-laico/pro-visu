`specimen` renders a looping clip of a typeface set as a fixed number of **left-aligned lines**
of glyphs whose letters and colours change over a composed sequence of **pulses**, then captures
it. The glyph size is derived from `lines` (the rows fill the top 80% of the frame); the bottom
20% shows the background and the font name. Point it at a font file and give it a name;
everything else has a sensible default. No `url` required.

```ts
{
  name: "brand-type",
  generator: "specimen",
  options: {
    font: "public/fonts/YourFont.woff2",
    name: "Your Font",
    template: "sweep",
  },
}
```

## Templates

`template` loads a named preset; your explicit options still override what it sets.

- **`"sweep"`** — a seamless-looping showcase of even per-character colour sweeps
  (muted → accent → foreground) on a dark palette chosen so the accent reads.
- **`"demo"`** — a labelled walkthrough of every pulse behaviour (each easing curve, a colour
  sweep, a mingle) with demo mode on; runs once, no mirror.

The **demo** template on a different typeface (a serif this time) — demo mode overlays each beat's
name as it plays, so you can see exactly what every pulse setting does:

## Pulses (the storyboard)

A **pulse** is one beat of the animation: a named span of time during which some fraction of the
glyph cells change letter and/or colour. An empty pulse is a hold. The clip length is the sum of
the pulse durations (doubled when `mirror` is on). Compose a sequence to author the whole clip.

export const pulseOptions = [
  { name: 'name', type: 'string', description: 'Human label for the beat (readability only). Default "".' },
  { name: 'durationMs', type: 'number', required: true, description: 'Length of the beat, in milliseconds. Required, must be positive.' },
  { name: 'chars', type: 'number (≥0)', description: 'Fraction of cells whose glyph changes during the beat (1 = every cell once; above 1 repeats cells). Default 0.' },
  { name: 'colors', type: 'number (≥0)', description: 'Fraction of cells whose colour changes during the beat (1 = every cell once; above 1 repeats cells). Default 0.' },
  { name: 'color', type: '"foreground" | "muted" | "accent"', description: 'Target every colour change at this token (a deliberate sweep) instead of a weighted-random pick. Omit for the default scattered recolour.' },
  { name: 'pacing', type: '"even" | "linear" | "ease-in" | "ease-out" | "ease-in-out" | "random"', description: 'How changes are distributed in time across the beat. Default "even". "linear"/"even" are uniform; "ease-in" front-loads; "ease-out" back-loads; "ease-in-out" bunches at both ends; "random" scatters.' },
]

<ConfigOptions items={pulseOptions} />

`pulses` must contain at least one beat. Omit it entirely and a built-in default storyboard
(`DEFAULT_PULSES` — a lively ~10-second outward half: intro hold → first letters → settle →
ripple → colour sweep → rest → quick burst → drift → finale → outro hold) is used, which
mirrors into a ~20-second seamless loop.

```ts
options: {
  font: "public/fonts/YourFont.woff2",
  name: "Your Font",
  pulses: [
    { name: "hold", durationMs: 800 },
    { name: "to accent", durationMs: 2200, colors: 1, color: "accent", pacing: "ease-in-out" },
    { name: "glyph drift", durationMs: 1400, chars: 0.5 },
  ],
}
```

## Colours

`colors` is the palette the glyphs cycle through (any CSS colour strings). Override any subset;
each key falls back independently.

export const colorOptions = [
  { name: 'background', default: '#eceef1', description: 'Backdrop behind the glyphs (and the bottom 20% band).' },
  { name: 'foreground', default: '#16181d', description: 'Primary glyph colour — the resting majority.' },
  { name: 'muted', default: '#a7adb6', description: 'Muted/secondary glyph colour.' },
  { name: 'accent', default: 'falls back to background', description: 'Accent colour for occasional pops. Left unset, accent glyphs blend into the backdrop — so set it (or a template) to make accent reads visible.' },
]

<ConfigOptions items={colorOptions} />

`colorWeights` sets the relative likelihood of each token on a **random** (non-targeted) recolour
— defaults `foreground: 2`, `muted: 2`, `accent: 1`, so accent is a rare pop. A weight of `0`
excludes that token from random recoloring entirely (an explicit pulse `color` can still target
it).

```ts
options: {
  font: "public/fonts/YourFont.woff2",
  name: "Your Font",
  colors: {
    background: "#1a1714",
    foreground: "#f6f3ed",
    muted: "#8a7e6f",
    accent: "#b49a77",   // set accent — left unset, accent glyphs blend into the background
  },
  colorWeights: { foreground: 3, muted: 2, accent: 1 }, // keep accent a rare pop
}
```

## Label

The top 80% of the frame is the glyph wall; the bottom 20% is a **gap** — plain background. The
`name` text lives in that gap, and `label` controls where it sits and how it looks. Position is
confined to the gap (the label moves around it, never over the glyphs).

export const labelOptions = [
  { name: 'anchor', type: '9 positions', default: '"bottom-left"', description: 'Corner/edge of the gap the label anchors to.' },
  { name: 'padding', type: 'number (px)', default: '32', description: 'Inset from the gap edges, all around. 0 = flush to the rendered corner.' },
  { name: 'size', type: 'number (0–1)', default: '0.22', description: 'Text size as a fraction of the gap height (scales with height).' },
  { name: 'weight', type: 'number (1–1000)', default: '500', description: 'Font weight.' },
  { name: 'color', type: 'CSS colour', default: 'falls back to foreground', description: 'Label text colour.' },
]

<ConfigOptions items={labelOptions} />

`anchor` is one of: `top-left`, `top-center`, `top-right`, `middle-left`, `middle-center`,
`middle-right`, `bottom-left`, `bottom-center`, `bottom-right`.

`padding` is measured to the **visible text** (the label's line box is trimmed to its cap height
and baseline), so the inset reads the same on all four sides — `padding: 0` puts the letters flush
to the rendered corner, with no phantom gap from the font's ascender/descender leading.

```ts
options: {
  font: "public/fonts/YourFont.woff2",
  name: "Your Font",
  label: {
    anchor: "bottom-right", // tuck it into the bottom-right of the gap
    padding: 8,             // nearly flush to the corner
    size: 0.28,             // a touch larger
    weight: 600,
    color: "#b49a77",
  },
}
```

## Behaviours

- **Width-stable layout.** Glyphs pack into `lines` rows using the font's real advance widths, and
  swaps are width-compensated (cancelling pairs + a per-line budget), so each line's right edge
  stays within `maxLineDrift` of its start — the wall shifts without the edges or line count ever
  jumping. A very small `characterPool` leaves less room to compensate.
- **Even-coverage picker.** Within a pulse, changes are distributed across cells so every glyph is
  touched once before any repeats. To wash the whole specimen to one colour evenly, set a pulse's
  `colors: 1` with `pacing: "even"` (and a `color` target) — that lands exactly one change on each
  glyph. A fraction below `1` touches that fraction of the cells.
- **Default glyph pool.** The specimen draws from
  `ABCDEFGHIJKLMNOPQRSTUVWXYZ0123456789$%&@#*+=` (uppercase A–Z, digits 0–9, and the symbols
  `$ % & @ # * + =`). Override it with `characterPool` (needs ≥2 distinct characters).
- **Blacklist.** `blacklist` removes glyphs from the pool case-insensitively (e.g. `"qxz"` and
  `"QXZ"` both drop Q, X, Z). A blacklist that would remove every glyph is ignored — the pool is
  never left empty.
- **Mirror.** With `mirror` on (default), the pulses play out and back as a palindrome that ends
  exactly on the opening frame, so the clip loops seamlessly. It doubles the clip length. Set
  `mirror: false` for a one-shot that ends on the last state.
