## Install

Two ways to run it — same CLI, same generators; they differ only in how you install it and author
config:

| | **Dev dependency** (recommended) | **Global / `npx`** (no install) |
|---|---|---|
| Install | `pnpm add -D auto-showcase` | `npm i -g auto-showcase`, or just `npx auto-showcase …` |
| Config | TS `showcase.config.ts` (`defineConfig`) — or JSON | JSON `showcase.config.json` |
| Editor help | Full TypeScript checking + autocomplete + hover docs | Autocomplete + validation + hover docs from a generated JSON Schema |
| Version | Pinned in your lockfile → you and CI build identical assets | Floating (npx fetches latest; pin with `auto-showcase@x.y.z`) |
| Best for | A repo you own, and CI pipelines | One-off captures, trying it out |

A TS `showcase.config.ts` needs the **dev dependency** because it does
`import { defineConfig } from "auto-showcase"`, which has to resolve from your project's
`node_modules` (both to type-check and to run). A JSON config has no import, so it works in any
mode — and `showcase init --json` generates a `showcase.schema.json` that gives editors the same
autocomplete, validation, and hover docs.

**A — As a project dev-dependency (recommended):**

```bash
pnpm add -D auto-showcase
```

Gives you the typed `defineConfig` authoring experience and pins the version in your lockfile, so
you — and CI — generate byte-identical assets.

**B — Without installing (npx or global):**

```bash
npx auto-showcase init --json     # dependency-free JSON config + JSON Schema
npx auto-showcase generate
```

…or install globally (`npm i -g auto-showcase`). No project dependency; the generated
`showcase.schema.json` still gives your editor autocomplete + validation on the JSON config.

> **Unreleased builds** (contributing, or pinning `main`): install from GitHub
> (`pnpm add -D github:chad-hill/auto-showcase`), a `pnpm link`, or a `file:` path — see the
> [package README](https://github.com/chad-hill/auto-showcase) for those.

## Initialise

```bash
npx showcase init
```

`init` is idempotent and sets up everything needed:

- writes a starter `showcase.config.ts` (unless a config already exists) — or, with `--json`, a
  `showcase.config.json` plus a `showcase.schema.json` for editor autocomplete,
- creates the output directory (`showcase/`) and adds it to `.gitignore`,
- adds a `"showcase": "showcase generate"` script to `package.json` (skip with `--no-script`),
- downloads the managed Chromium and ensures ffmpeg (skip with `--skip-browser`).

The first browser download is one-time, cached, and shared across projects.

## Configure

Open `showcase.config.ts` and point an asset at your site. The starter looks like this:

```ts
import { defineConfig } from "auto-showcase";

const URL = "http://localhost:3101";

export default defineConfig({
  settings: {
    outDir: "showcase",
    concurrency: 2,
    browser: { headless: true },
    defaults: {
      "scroll-reel": { width: 1440, height: 900, fps: 30 },
    },
  },
  assets: [
    { name: "home-reel", url: URL, generator: "scroll-reel" },
  ],
});
```

If you scaffolded with `--json`, the equivalent `showcase.config.json` carries a
`"$schema": "./showcase.schema.json"` line — your editor reads it for the same autocomplete and
validation. Refresh that schema after upgrading the tool with `showcase schema`.

See [Configuration](/docs/configuration) for the full shape, and
[Generators](/docs/generators) for every asset type and its options.

## Generate

Start your site (or use a deployed URL), then:

```bash
npx showcase generate
```

Each asset is rendered into `outDir` and recorded in `manifest.json`. Review what was
produced with:

```bash
npx showcase list
```

> **Don't want to start the site yourself?** Configure a [managed server](/docs/configuration/managed-server)
> and the CLI will build → start → capture → stop it for you, so `showcase generate` is the only command you run.

## Iterate faster

- `showcase generate --draft` — lower fps/scale and a faster encoder for quick previews.
- `showcase generate --asset home-reel` — only (re)build named assets (repeatable).
- `showcase generate --cache` — skip assets whose inputs + options are unchanged.
- `showcase generate --skip-build` — keep the managed server but skip its build step.

See the full [CLI reference](/docs/cli).

## Requirements

- **Node.js** (the CLI runs on Node).
- **Chromium** — Playwright's managed build, fetched automatically; or point at your own via
  `settings.browser.channel` / `executablePath`.
- **ffmpeg** — a working binary is ensured automatically for the video generators.
